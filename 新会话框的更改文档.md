# 新会话框的更改文档

## 变更范围

- `backend/api/session.py`
  - 新增 `PATCH /api/session/<session_id>/name`。
  - 该接口只读取会话数据、校验 `session_name`、写回 `session_data["session_name"]`。
  - 不修改 `messages`、`memories`、`config`、`user_name`、`character_name`、RAG、向量索引或聊天生成逻辑。

- `backend/web/static/js/chat.js`
  - 新增会话名长度常量、校验函数、轻量命名弹窗、`renameSession()`、`promptRenameSession()`。
  - 点击左侧 `+ 新对话` 时，先通过自定义弹窗输入会话显示名，再创建会话并保存 `session_name`。
  - 会话列表中为每个会话新增轻量重命名入口。
  - 会话列表中为每个会话新增轻量删除入口，调用项目已有 `DELETE /api/session/<session_id>`。
  - 删除当前会话后自动切换到剩余第一条会话；如果没有剩余会话，则创建一个空的新会话。
  - 点击重命名入口会阻止事件冒泡，不触发切换会话。

- `backend/web/static/css/style.css`
  - 追加会话信息布局、角色副标题省略、重命名按钮 hover/focus 样式。
  - 追加会话删除按钮 hover/focus 样式。
  - 追加专用会话命名弹窗样式，弹窗位于屏幕正中央。
  - 未改变侧栏宽度、聊天主区域、顶部栏、输入框或其他页面布局。

## 新增接口

`PATCH /api/session/<session_id>/name`

请求体：

```json
{
  "session_name": "新的会话名称"
}
```

成功返回：

```json
{
  "success": true,
  "session_id": "...",
  "session_name": "新的会话名称"
}
```

校验规则：

- 自动去除首尾空格。
- 空名称返回 400。
- 超过 40 个字符返回 400。

## 交互说明

- 点击左侧 `+ 新对话` 后弹出项目内自定义命名框。
- 输入有效名称后创建新会话，并通过新增接口保存到 `session_name`。
- 已有会话 hover 时显示重命名入口。
- 已有会话 hover 时显示删除入口。
- 命名框支持 Enter 确认、Esc 取消、点击遮罩取消。
- 重命名当前会话后同步 `State.sessionName` 和当前会话元信息。
- 重命名完成后刷新左侧会话列表。
- 删除会话使用已有删除接口，该接口会按项目原逻辑删除会话文件并同步删除对应会话向量索引。

## 隔离说明

本次变更只管理会话显示名称，即 `session_name`。

未改动：

- `user_name`
- `character_name`
- 角色配置
- system prompt
- messages
- memories
- RAG 召回
- 向量索引
- 聊天生成
- `/api/history`
- `/api/chat/config`

## 后续补充

- 命名弹窗已从偏左调整为屏幕正中央。
- 删除会话入口只增加在左侧会话列表，不改聊天生成、RAG、角色配置或参数配置。
