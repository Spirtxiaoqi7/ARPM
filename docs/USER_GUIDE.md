## 快速入门

### 3分钟上手

1. **启动系统**
   ```bash
   # Windows
   start.bat
   
   # 或命令行
   python app.py
   ```

2. **访问界面**
   - 浏览器打开 http://localhost:5000

3. **首次配置**
   - 点击右上角"设置"
   - 输入API密钥 (从DeepSeek或其他服务商获取)
   - 点击"测试连接"确认配置正确
   - 点击"保存"

4. **开始对话**
   - 在输入框输入问题
   - 按Enter或点击发送按钮

## 系统配置

### API配置

**必填项**:
- **API密钥**: 你的LLM服务密钥 (如 `sk-xxxxxxxxxx`)
- **接口地址**: API基础URL
  - DeepSeek: `https://api.deepseek.com`
  - OpenAI: `https://api.openai.com`
- **模型名称**: 
  - `deepseek-chat` (推荐)
  - `gpt-4`
  - `gpt-3.5-turbo`

**可选项**:
- **系统提示词**: 定义AI角色性格和行为
  ```
  示例: 你是温柔体贴的虚拟伴侣，擅长倾听和安慰。
  ```

### 检索参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 分块大小 | 文档切分长度(字符) | 600 |
| 检索数量 | 每次检索的片段数 | 5 |
| 衰减率 | 记忆时间权重 | 20 |

**分块大小建议**:
- 小说/故事: 800-1000 (保留上下文)
- 问答/知识: 400-600 (精确检索)
- 聊天记录: 600-800 (平衡)

## 对话流程

### ARPM两轮协议

**第1轮 - 分析阶段**:
```
用户: 你是谁？
系统: [内部检索+分析]
显示: "首轮分析完成，背景知识已存入记忆。请继续第二轮对话获取完整回复。"
```

**第2轮+ - 输出阶段**:
```
用户: 你的爱好是什么？
系统: [检索+结合历史记忆]
显示: 完整回复，融入角色设定和上下文
```

**为什么要两轮？**
- 第1轮静默存储用户偏好和背景
- 第2轮起AI了解足够信息，回复更连贯
- 避免首轮"不了解情况"的尴尬

## 知识库管理

### 上传文档

1. 打开设置 → 知识库区域
2. 点击"选择文件"
3. 选择 .txt / .md / .json 文件
4. 等待自动分块和上传
5. 查看"已加载 X 个片段"

### 管理知识片段

点击"管理"按钮进入知识库管理界面:

**左侧统计面板**:
- 📚 全部片段
- 🔒 时间锁定
- 📜 怀旧模式
- 📋 条件激活
- 🔑 关键词提升
- 🎬 场景记忆

**搜索功能**:
- 关键词搜索: 输入文字快速过滤
- RAG检索: 语义相似度搜索
- 类型筛选: 按标记类型过滤

**分页控制**:
- 支持 20/50/100/200 条/页
- 大数量时建议50或100条/页

### 标记说明

**🔒 时间锁定**:
- 不受时间衰减影响
- 适用于核心人设信息
- 点击🔒按钮切换

**📜 怀旧模式**:
- 越久远的记忆权重越高
- 适用于背景故事
- 可设置怀旧因子(0.01-0.1)

**📋 条件激活**:
- 设置触发规则
- 支持关键词/轮次范围/正则
- 条件满足时才检索

**🔑 关键词提升**:
- 设置关键词和权重(1.0-3.0)
- 匹配时提升检索得分
- 多关键词有防滥用衰减

**🎬 场景记忆**:
- 将记忆关联到场景
- 场景内衰减重置
- 支持剧情段落管理

## 消融测试

用于对比测试各组件效果。

### 开关说明

**ARPM系统总开关**:
- 关闭 → 纯LLM对话，无RAG增强
- 开启 → 完整RAG增强
- 用途: 对比有无检索增强的差异

**BM25+混合检索**:
- 关闭 → 仅使用向量语义检索
- 开启 → 向量+关键词双重检索
- 用途: 测试关键词匹配的重要性

**CoT重排序**:
- 关闭 → 直接输出，无思维链分析
- 开启 → LLM先分析再回复
- 用途: 测试推理过程对结果的影响

**时间感知衰减**:
- 关闭 → 所有记忆等权重
- 开启 → 新记忆权重高，旧记忆衰减
- 用途: 测试时间因素对长期对话的影响

**关键词提升**:
- 关闭 → 忽略关键词标记
- 开启 → 应用关键词加权
- 用途: 测试关键词干预效果

### 测试建议

1. **基线测试**: 全部开启，记录效果
2. **逐个关闭**: 每次关闭一个组件，观察差异
3. **极端测试**: 全部关闭，对比纯LLM
4. **组合测试**: 尝试不同的开关组合

## 消息反馈

### 点赞 (👍)

- 表示喜欢这类回复风格
- 用于系统学习用户偏好
- 不立即改变当前AI行为

### 差评 (👎)

- 表示不喜欢这类回复
- 帮助系统了解避免的倾向
- 用于长期偏好调整

### 举报 (🚫)

**使用场景**:
- 内容有害或不当
- 事实错误严重
- 涉及隐私泄露
- 与设定不符

**举报流程**:
1. 点击🚫举报按钮
2. 选择举报原因
3. 可选填写详细说明
4. 点击"提交举报"
5. 系统自动:
   - 归档原内容
   - 重新生成安全回复
   - 标记"已重新生成"

## 聊天记录管理

### 单条删除

1. 鼠标悬停在消息上
2. 显示操作按钮
3. 点击🗑️删除
4. 确认删除

### 清空聊天记录

1. 点击底部"🗑️ 清空记录"
2. 查看确认弹窗信息:
   - 当前会话名称
   - 消息数量
3. 点击"确认清空"或"取消"

**注意**: 
- 清空后不可恢复
- 清空不影响知识库
- 自动重置切分状态

### 自动归档

**触发条件**:
- 自动切分开关开启
- 未归档的AI回复超过10K字符
- 距离上次归档超过60秒

**行为**:
- AI回复自动存入知识库
- 聊天记录保持不变
- 显示归档提示

## 故障排查

### 启动问题

**错误: 找不到模块**
```bash
# 重新安装依赖
pip install -r requirements.txt
```

**错误: 端口被占用**
```bash
# 修改端口
set PORT=5001
python app.py
```

### API问题

**连接测试失败**:
1. 检查API密钥格式 (以sk-开头)
2. 检查接口地址是否包含https://
3. 检查网络连接
4. 查看防火墙设置

**回复为空或报错**:
1. 检查模型名称是否正确
2. 检查API余额是否充足
3. 检查API服务状态

### 知识库问题

**上传失败**:
1. 检查文件格式 (.txt/.md/.json)
2. 检查文件大小 (<5MB)
3. 检查文件编码 (UTF-8)

**检索无结果**:
1. 检查知识库是否为空
2. 检查查询内容是否相关
3. 尝试降低检索阈值

### 性能问题

**响应慢**:
1. 减少检索数量 (top_k: 5→3)
2. 减少分块大小 (600→400)
3. 关闭部分消融测试组件

**内存不足**:
1. 重启服务释放内存
2. 减少同时处理的会话数
3. 清理历史知识库

## 高级用法

### 角色设定模板

```
【角色名称】: 林小满
【性格特征】: 温柔体贴，略带撒娇，会关心人
【说话风格】: 口语化，偶尔用语气词"呀~""呢"
【背景设定】: 与用户是恋人关系
【禁忌】: 不说"我爱你"，用其他方式表达
```

### 知识库优化

**提高检索准确性**:
1. 为核心信息添加🔒时间锁定
2. 为背景故事设置📜怀旧模式
3. 添加相关🔑关键词
4. 合理设置📋条件激活规则

**避免检索污染**:
1. 无关文档单独存储
2. 使用条件激活隔离场景
3. 定期清理测试数据

### 系统诊断

设置面板 → 🔍 系统诊断:

**诊断项**:
- 向量数据库完整性
- 元数据一致性
- FAISS索引有效性
- 模型加载状态
- 磁盘空间
- 会话文件完整性
- 孤立chunk检测
- 场景数据完整性

**自动修复**:
- 修复数据不一致
- 重建损坏索引
- 清理孤立数据
