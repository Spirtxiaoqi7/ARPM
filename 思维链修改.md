# 思维链修改

## 思维链修改日志

更新时间：2026-05-11

本次修改集中在前端 `backend/web/static/js/chat.js`，目标是修复切换聊天/研究风格或刷新页面后会话状态丢失，以及协议修复占位文本被误显示在“思维链 / 协议诊断”面板中的问题。

### 修改摘要

1. 新增 `arpm.ui.current_session` 本地存储 key，用于记录当前正在使用的 `sessionId`。
2. 新增 `getSavedCurrentSessionId()` 与 `saveCurrentSessionId()`，统一处理当前会话读取和保存，失败时只记录前端警告，不影响正常聊天。
3. 在加载历史会话、创建新会话、首次发送消息生成会话、消息发送成功后，都会保存当前 `sessionId`，避免刷新或切换风格后回不到原会话。
4. 初始化时优先恢复上次保存的会话；如果该会话已不存在，则回退到会话列表中的第一个会话。
5. 新增 `isProtocolRepairPlaceholder()`，识别并过滤 `协议修复：原输出缺少显式分析` 这类占位内容，兼容纯文本、带 `<analysis>` 标签、有无句号和空白差异的形式。
6. 调整 `renderAnalysis()`：正常分析内容仍显示；只有协议修复占位内容时显示为空状态，避免用户把占位诊断误认为真实思维链或系统错误。

### 影响范围

- 修改范围：仅前端展示和浏览器本地状态恢复。
- 未修改：后端 API、RAG 检索、知识库逻辑、会话数据结构、生成算法、模型参数、协议修复算法。
- 用户可见变化：刷新页面或切换风格后更稳定地回到当前会话；思维链面板不再展示协议修复占位文本。

### 1. 新增当前会话本地存储 key

修改位置：`backend/web/static/js/chat.js` 顶部 `STORAGE_KEYS`

```js
const STORAGE_KEYS = {
    api: "arpm.api.config",
    ui: "arpm.ui.config",
    currentSession: "arpm.ui.current_session"
};
```

错误修复权重：`中`

功能优化：刷新页面、切换风格后仍能知道上次正在使用哪个会话，减少“必须新建对话才能继续”的情况。

### 2. 新增当前会话读取和保存函数

修改位置：`backend/web/static/js/chat.js`，`saveUiConfig()` 后面

```js
function getSavedCurrentSessionId() {
    try {
        return window.localStorage.getItem(STORAGE_KEYS.currentSession) || "";
    } catch (error) {
        console.warn("Read current session failed", error);
        return "";
    }
}

function saveCurrentSessionId() {
    if (!State.sessionId) {
        return;
    }
    try {
        window.localStorage.setItem(STORAGE_KEYS.currentSession, State.sessionId);
    } catch (error) {
        console.warn("Save current session failed", error);
    }
}
```

错误修复权重：`中`

功能优化：把会话恢复能力集中在两个前端函数里，失败时只打印警告，不影响聊天、RAG 或保存功能。

### 3. 增强协议修复占位文本过滤

修改位置：`backend/web/static/js/chat.js`，`renderAnalysis()` 前

```js
function isProtocolRepairPlaceholder(text) {
    const normalized = String(text || "")
        .replace(/<\/?analysis>/gi, "")
        .replace(/[\s\u3000]+/g, "")
        .replace(/[。．.]+$/g, "");
    return normalized === "协议修复：原输出缺少显式分析";
}
```

错误修复权重：`高`

功能优化：可以同时识别以下几类协议修复占位内容，避免它们被显示成真实思维链：

- `协议修复：原输出缺少显式分析。`
- `协议修复：原输出缺少显式分析`
- `<analysis>协议修复：原输出缺少显式分析</analysis>`
- `<analysis> 协议修复：原输出缺少显式分析。 </analysis>`

### 4. 调整思维链渲染逻辑

修改位置：`backend/web/static/js/chat.js`，`renderAnalysis()`

```js
function renderAnalysis(analysis, protocolInfo = null) {
    const rawDisplayAnalysis = protocolInfo?.original_analysis || analysis;
    const displayAnalysis = isProtocolRepairPlaceholder(rawDisplayAnalysis) ? "" : rawDisplayAnalysis;
    const showProtocolBlock = protocolInfo && !isProtocolRepairPlaceholder(rawDisplayAnalysis);
    const protocolBlock = showProtocolBlock ? `
        <div class="rag-item" style="margin-bottom:10px;">
            <div class="rag-header">
                <span class="rag-source">协议诊断</span>
                <span class="rag-score">${protocolInfo.resolved_model_mode || "-"}</span>
            </div>
            <div class="rag-meta">
                analysis_tag=${protocolInfo.has_analysis_tag} · response_tag=${protocolInfo.has_response_tag}
                · visible_reply=${protocolInfo.visible_reply} · thinking_only=${protocolInfo.looks_thinking_only}
                · repaired=${protocolInfo.was_repaired ?? false} · needs_repair=${protocolInfo.needs_repair}
            </div>
        </div>
    ` : "";
    if (!displayAnalysis && !protocolBlock) {
        DOM.cotContent.innerHTML = '<div class="empty">当前轮没有额外分析</div>';
        return;
    }
    DOM.cotContent.innerHTML = `${protocolBlock}<div class="rag-item"><div class="rag-text">${Utils.formatText(displayAnalysis || "无显式分析内容")}</div></div>`;
}
```

错误修复权重：`高`

功能优化：正常有分析内容时仍展示；只有协议修复占位内容时显示为空状态，用户不会误以为系统报错或把协议文本当作真实思维链。

### 9. 初始化时优先恢复上次当前会话

修改位置：`backend/web/static/js/chat.js`，`init()`

```js
const sessions = await loadSessions();
if (sessions.length > 0) {
    const savedSessionId = getSavedCurrentSessionId();
    const restoredSession = sessions.find((item) => item.id === savedSessionId);
    await loadHistory((restoredSession || sessions[0]).id);
    await loadSessions();
} else {
    createNewSession();
}
```

错误修复权重：`高`

功能优化：刷新页面后优先回到用户刚才使用的会话；如果该会话被删除，再回退到第一个会话，避免页面卡住或必须手动新建对话。

## 针对本次问题的原因说明

右上角从“聊天风格”切换到“研究风格”只会切换页面 class 和本地 UI 配置，不会改变协议参数、RAG 参数或生成算法。

本次出现：

```text
standard
analysis_tag=true · response_tag=true · visible_reply=true · thinking_only=false · repaired=true · needs_repair=false
<analysis>协议修复：原输出缺少显式分析</analysis>
```

原因是后端协议修复已正常完成，但前端过滤条件原先只识别带中文句号的纯文本：

```text
协议修复：原输出缺少显式分析。
```

没有识别带 `<analysis>` 标签、无句号的版本。本次已通过第 3 条修改修复。
